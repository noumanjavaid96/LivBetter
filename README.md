# Excel to Notion CSV Converter

## Project Overview

This project is designed to convert a complex, multi-sheet Excel file named `Provider Services per Location (A&A + RISE) (1).xlsx` into a single, consolidated CSV file that is optimized for import into Notion. The source Excel file contains a variety of data structures, including tables, lists, and unstructured notes, spread across 30 different sheets.

The primary goal of this project is to create a unified, clean, and user-friendly dataset that can be easily managed and viewed in a database tool like Notion.

## Files in this Project

-   **`Provider Services per Location (A&A + RISE) (1).xlsx`**: The original, multi-sheet Excel file that serves as the data source.
-   **`create_notion_csv.py`**: A Python script that reads all the sheets from the Excel file, intelligently extracts and consolidates the data, and outputs a single, clean CSV file.
-   **`notion_database.csv`**: The final, consolidated CSV file that is ready to be imported into Notion. This file contains the structured data from all 30 sheets of the original Excel file.
-   **`.gitignore`**: A file that tells Git to ignore the `processed_sheets` directory, which may be created during intermediate steps.

## How to Use

### Prerequisites

-   Python 3.x
-   Pandas library (`pip install pandas`)

### Running the Conversion

To generate or regenerate the `notion_database.csv` file, simply run the `create_notion_csv.py` script from your terminal:

```bash
python create_notion_csv.py
```

This will process the `Provider Services per Location (A&A + RISE) (1).xlsx` file and create the `notion_database.csv` file in the same directory.

### Importing to Notion

1.  Open your Notion workspace and create a new, empty page.
2.  Click on the `Table` option to create a new database.
3.  In the new, empty table, click on the `...` menu in the top-right corner.
4.  Select the `Merge with CSV` option.
5.  Upload the `notion_database.csv` file generated by the script.

Notion will automatically import the data and create the necessary columns. You can then customize the database views, filters, and sorts as needed.

## Output Format

The `notion_database.csv` file has the following columns:

-   **`Clinic`**: The name of the clinic or location.
-   **`Provider`**: The name of the healthcare provider.
-   **`Category`**: A category to help organize the data, such as "Provider Services," "Scheduling/Service Summary," or "Previous Office Info."
-   **`Item`**: The specific item being described, such as "Service Matrix" or the name of the original sheet.
-   **`Details`**: A JSON-formatted string containing the detailed information related to the item. This is designed to be easily readable and provides a structured view of the data.
-   **`Additional Info`**: A column for any additional notes, context, or information that doesn't fit into the other columns, ensuring that no data is lost.
